## Notes / decisions

- The BEAM error branch is placed between Go and Ruby in the elif chain.
  Go's `panic:` prelude and the BEAM `** ` prelude never overlap, but
  the placement keeps the systems-language group (Go, Rust, BEAM)
  clustered together for readability. Verified no regression in any
  other framework via the full error test file (73 passing).
- The DOI regex was initially non-greedy with a lookahead that stopped
  at the first `.` — fixed to greedy + trailing-punctuation trim
  because real DOIs (`10.1145/3372297.3417883`) contain dots in their
  suffix.
- The order_number value regex's alternation initially put the 1-char
  fallback first; regex matched the short form and truncated
  `ABC-12345` to `A`. Flipped the alternation so the longer
  bookend-bounded form is tried first.
- `_extract_statuses` initially iterated matchers and appended in
  matcher order; this put Read entries before Delivered even when
  Delivered came first in the source. Switched to collect-then-sort-
  by-offset.
- All five features wrote clean (no ruff fixup amends needed). The
  pattern of running ruff baseline first + writing each file with the
  documented test-file conventions (no blank line between docstring
  and `from __future__`, use `/var/log/...` not `/tmp/...` in tests)
  paid off this tick.
