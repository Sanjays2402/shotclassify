## Theme

Cross-category enrichment dominated the tick: three of the five features add
new `raw[...]` keys that run on every category's OCR text (MAC, timezone,
credit-card). One advances the privacy / compliance surface (address redact
mode is now in the same allow-list as email / phone / SSN / IBAN). One is a
pure code-snippet quality-of-life win (line-number gutter detection) that
also makes every downstream code detector more accurate by feeding it the
de-numbered body.

## Notes

- The `credit_card` redact mode in shotclassify_common.redact already existed
  and was paired this tick with the new `credit_cards` extractor. The
  extractor surfaces BIN+last4 metadata; the redactor swaps the raw PAN with
  `[REDACTED:credit_card]` before persistence. Together they let an operator
  know what card was used without ever storing the secret. The extractor's
  test suite includes a security guarantee test
  (`test_full_pan_never_in_output*`) that asserts no full PAN string ever
  appears in the returned dicts -- this guarantee is independent of the
  redact mode.
- `phone` redact mode was on the backlog (#56) but already exists in
  redact.py. Re-classified the backlog item as "refine to `<PHONE>` stub
  form" so we don't accidentally re-ship the existing behaviour.
- Line-numbering detection runs FIRST in `enrich_code` because every
  downstream detector wants to see the actual code, not the gutter. This
  reordering doesn't break any existing test because unnumbered snippets
  pass through `detect_numbered` unchanged.
- IPv6 masking in the MAC extractor: I considered a more elaborate IPv6
  parser but kept it simple -- the mask only needs to identify spans that
  could fool the MAC matcher, and a compressed `::` or full 8-group form
  is enough. The MAC matcher's word-boundary lookarounds catch the rest.
- Numeric timezone offsets de-dup intelligently: `+0530` and `+05:30`
  collapse to `+05:30`; `-08` and `-0800` collapse to `-08`. Two-digit
  hour padding is always applied so `+5` never appears in storage.
