## Shipped (5 commits + bootstrap on feature/autoship)

1. **0d85454** `feat(extract/receipt): tip and gratuity extraction`
   - Adds `ReceiptFields.tip` (and `payment_method` placeholder used
     by feature 2).
   - Parser recognises Tip / Gratuity / Service charge / bare Service
     lines, last-occurrence wins so a "Tip 15% = 3.00" suggestion
     table never beats the line the customer paid.
   - Items parser now skips tip / gratuity / service lines so the
     gratuity is no longer also reported as a $4.00 line item.
   - 8 tests in tests/test_receipt_tip_extraction.py.

2. **9ac3b34** `feat(extract/receipt): payment method detection`
   - Canonical enum (visa, mastercard, amex, discover, apple_pay,
     google_pay, debit, credit, cash) ordered most-specific first so
     "Apple Pay (Visa)" tags apple_pay and "Credit Card AMEX 1001"
     tags amex.
   - Word-boundary guards keep "Cashback" / "Cashier" from triggering
     the cash tag.
   - 20 parametrised + standalone tests in
     tests/test_receipt_payment_method.py.

3. **f36757d** `feat(extract/error): Go panic and Ruby/Rails stacktrace support`
   - New `go` framework: parses panic line into exception + message
     (treating "runtime error" / "fatal error" as the exception type),
     pulls the topmost *.go:LINE frame, likely-cause hints for nil
     pointer deref, index-out-of-range, concurrent-map, send-on-closed
     channel, and goroutine deadlock.
   - New `ruby` framework: ``file.rb:LINE:in `method'`` frames (incl.
     ``\\tfrom ...`` continuation), ``(ClassName)`` exception trailer
     supporting namespaced names that don't end in Error/Exception
     (``ActiveRecord::RecordNotFound``), likely-cause hints for
     NoMethodError / NameError / ArgumentError / LoadError /
     ActiveRecord::RecordNotFound.
   - Branch ordering puts Go/Ruby AFTER Python/Node/JVM (regression
     test ensures Python with "goroutine" in a string literal still
     classifies as Python).
   - 9 tests in tests/test_error_go_ruby_extraction.py.

4. **1afe733** `feat(extract/code): more languages + framework detection`
   - Adds language hints for Kotlin, Swift, C#, Elixir, Scala, PHP,
     Haskell on top of the existing 10 languages.
   - Switched `_FAST_HINTS` from a dict to an ordered tuple so the
     precedence rules are explicit (SQL ahead of Python so SELECT a
     FROM t doesn't lose to Python's bare `from `; shell ahead of
     PHP so `echo $f` doesn't lose to PHP's `echo `; etc.).
   - New `detect_framework(code)` returns react / vue / angular /
     nextjs / django / flask / fastapi / rails / spring / express /
     gin / actix / tokio / None.
   - 34 tests in tests/test_code_language_and_framework.py.

5. **48a349c** `feat(redact): JWT, AWS, GitHub, Slack token redaction modes`
   - Four new PII modes added in lockstep across `redact._PATTERNS`
     and `tenant_settings.PII_REDACT_MODES`. The lockstep is asserted
     by a test so a future code change cannot silently drift one side.
   - Patterns are tight against false positives: AWS AKIA/ASIA + 16
     base32, GitHub published prefixes, Slack xox{a-z}- family, JWT
     requires eyJ header.
   - 26 parametrised tests in tests/test_redact_developer_secrets.py.
