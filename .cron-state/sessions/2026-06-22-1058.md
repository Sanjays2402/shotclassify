1. **f4a089b feat(extract/chat): replied-to / quoted-message detection into ChatFields.quotes**

   `ChatFields.quotes` -- list of `{sender, quoted_sender, quoted_text, reply_text}` dicts.

   Three shapes:
   - line-leading `> quoted text` runs (Slack / IRC / email / Discord)
   - `Replying to <name>: <body>` preamble (iMessage / WhatsApp / Telegram / Discord)
   - `> Sender: text` Slack attribution-inside-quote
   - Discord `> @user body` reply-mention

   Hardest part: ordering the preamble matcher BEFORE the transcript-sender
   matcher so "Replying to Alice" doesn't tag as a speaker named "Replying to
   Alice". Also blank-line-then-`>`-run terminator semantics for breaking
   consecutive `>` runs into distinct quote blocks.

   False-positive defences: rejects `->` arrows, `=>` JS arrows, `</div>`
   close tags, stray `>` with no quoted body.

   37 tests, cap 20 entries, dedupe on quadruple, LLM wire format updated.

2. **f616880 feat(extract/chat): voice / image / video / file attachment marker detection into ChatFields.attachments**

   `ChatFields.attachments` -- list of `{kind, sender?, duration?, name?}` dicts.

   Three shapes:
   - WhatsApp/iMessage bracketed `[Image]` / `[Voice note 0:23]` / `[Document: file.pdf]`
   - Telegram emoji-prefixed `📷 Photo` / `🎤 Voice (0:42)` (26-emoji curated catalogue)
   - Generic English line-anchored `Voice message (0:42)` / `Video call · 1m 23s` / `Missed video call`

   Canonical lowercase kind vocabulary: image/video/voice/audio/document/sticker/
   gif/location/contact/video_call/audio_call. Duration in MM:SS / H:MM:SS /
   Nm Ms forms.

   Key defences:
   - bracketed labels must be in alias map (rejects `[issue-123]` / `[TICKET-99]`)
   - English shape requires line-anchoring + closed-vocabulary label list
     (rejects prose "I voiced my opinion")
   - leading `Sender: ` transcript prefix stripped before English match so
     transcript-attached attachment lines still tag

   Match ordering bracket->emoji->English with span-overlap-rejection so no
   double-tagging.

   61 tests, cap 30, dedupe on quadruple, LLM wire format updated.

   New extractor module `error_fingerprints.py`. Output: list of
   `{vendor, kind, id}` dicts.

   7 vendor catalogues:
   - Sentry (full 32-hex + short bracketed + inline short)
   - Datadog (`dd.trace_id=` + `dd.span_id=` + bare-trace-id with DD anchor)

   Anchoring philosophy: every short/numeric form REQUIRES vendor keyword on

   Hex IDs lowercased for stable dedupe, alphanumeric IDs case-preserved
   (Bugsnag `ABCD1234` round-trips intact).

   43 tests, cap 30, runs cross-category in pipeline.

5. **4214941 feat(redact): bank-account / routing-number redaction mode**

   New `bank_account` mode added to `PII_REDACT_MODES` allow-list.

   Captures routing + account number pairs from check MICR encoding lines and
   ACH/wire setup screens. Recognised labels:
   - routing side (9-digit): Routing / Routing No / Routing Number / Routing # /
     ABA / ABA Routing # / RTN
   - account side (6-17 digit): Account / Account No / Account Number /
     Account # / Acct / Acct No / A/C / A/C No / Acc

   Key safety properties:
   - matcher REQUIRES label before digits (bare 9-digit / phone / order /
     SSN / passport all left UNCHANGED)
   - 18+ digit runs left UNCHANGED via trailing word-boundary failure
     (partial 17-of-18 redaction would leak trailing digit, wrong direction
     for a privacy mode)
   - custom `_sub_bank_account` substitution preserves the label so reader
     knows the field WAS banking data without the number leaking

   47 tests, mirrors passport/drivers_license substitution pattern.
