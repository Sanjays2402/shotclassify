## What shipped

1. **Code framework expansion** — Adds five popular backend frameworks
   the original catalog missed: Laravel (Illuminate facades), Symfony
   (`#[Route]` + `AbstractController`), Phoenix
   (`Phoenix.Router`/`pipe_through`), Quarkus (`io.quarkus`), Micronaut
   (`io.micronaut`). Quarkus/Micronaut iterate BEFORE Spring so a
   compat shim that uses `@RestController` still tags the more-specific
   runtime. 14 new test cases + regressions for the existing set.

2. **Receipt currency inference** — Widens `_detect_currency` beyond
   the four-symbol set ($/EUR/GBP/JPY) to recognise 20 ISO codes:
   USD, EUR, GBP, JPY, CAD, AUD, NZD, CHF, SEK, NOK, DKK, INR, MXN,
   BRL, ZAR, SGD, HKD, CNY, RMB (normalised to CNY), KRW. ISO codes
   match with regex word boundaries. Last match wins so a tourist
   receipt's closing "Total in CAD" beats a header "USD". `$` with no
   ISO code still falls back to USD. 17 new tests covering each code,
   header/total precedence, embedded-letter rejection, and the
   existing symbol paths.

3. **HTTP status classifier** — Adds an HTTP branch to `parse_error_text`
   that fires AFTER all language stacktrace branches (Python / Node /
   JVM / Go / Ruby) and BEFORE the generic Error/Exception fallback,
   so a Python traceback that mentions "HTTP 500" in a string literal
   still classifies as python. Recognises `HTTP/1.1 500 ...`,
   `HTTP 404 Not Found`, `Response: 403 Forbidden`, `status: 503`,
   and bare `<code> <reason>` pairs for well-known phrases. Tags
   `framework="http"`, `exception="HTTP <code>"`, `message=<reason>`,
   and `likely_cause=<per-code hint>` for the 13 most common codes
   plus a class label for the rest. New `parse_http_status` helper
   exposed at the package level. 18 new tests including stacktrace
   regression coverage.

4. **Receipt line-item quantity inference** — Extends `_parse_items`
   to recognise quantity-prefixed and trailing-`@` line shapes:
   `2 x Latte 6.00 = 12.00`, `3 X Espresso 3.50`, `2 * Croissant 3.00`,
   `Latte 2 @ $6.00`. When both unit + extended totals are printed
   we store the UNIT price so qty * price reconstructs the extended
   total cleanly. Decimal quantities (`1.5 x Apples`) and comma
   decimals (`6,00`) supported. Backward compatible: lines without a
   quantity marker still fall through to the original `desc price`
   regex with qty=None. 13 new tests.

5. **Cross-category file-path extractor** — Mirrors last tick's URL
   extractor. Catches POSIX absolute, home-relative (`~/...`),
   Windows drive (`C:\\...`), and UNC (`\\\\server\\share\\...`)
   paths. URL spans are masked before scanning so a URL's path
   component never double-counts under both `raw["urls"]` and
   `raw["paths"]`. Overlapping matches dedup by span (POSIX inside
   home-relative cannot leak). Output capped at 50. New
   `extract_paths` helper at the package level + pipeline
   integration for every category. 28 new tests.
