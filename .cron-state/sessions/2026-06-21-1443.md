## Risks / followups

- Multi-holder on the SAME line in copyrights only surfaces the first holder. Documented in test docstring + STATE.md notes. Real headers print one per line so this only affects pathological OCR.
- Docstring detector pulls a `/** ... */` block buried inside a function body as the top-level docstring when no other JSDoc precedes it. Accepted trade-off because we don't track indentation depth.
- Airport extractor's `_NEVER_AIRPORTS` set is a moving target — real-world prose will surface more 3-/4-letter acronyms that should be rejected. The catalogue is the safety net for genuine codes that get rejected by accident.
- Social extractor's Twitter `@handle` matcher with `follow me`/`find me` anchor is intentionally loose so it catches "Follow me on X at @jack" style prose. A future tighter mode could require the anchor to be a hard prefix.
