## Features

### 1. cross-category phone-number extractor — 1c6a05f
`raw["phones"]`. E.164 with mixed separators (parens, dashes, dots,
spaces), NANP-formatted with matched separators + 2..9 leading
digits on area & exchange, keyword-prefixed bare NANP (Phone / Tel
/ Mobile / Cell / Fax). Output: canonical digits-only with `+`
preserved for E.164. E.164 hits also register the trailing 10-digit
form so NANP duplicates of the same number get suppressed in the
output. Cap 50. 42 tests.

### 2. cross-category UUID extractor — 8373e6b
`raw["uuids"]`. Dashed RFC 4122 (8-4-4-4-12 hex) + compact 32-hex
form, version-nibble enforced (1..5), variant nibble intentionally
unconstrained so Microsoft GUIDs land. Non-hex boundaries on both
ends prevent biting longer SHA-1 / SHA-256 hashes. Canonical
lowercase + hyphenated output so dashed and compact forms of the
same UUID collapse. "Nil" all-zero UUID rejected as placeholder.
Cap 50. 34 tests.

### 3. cross-category git SHA extractor — 81533f8
`raw["git_shas"]`. Full SHA-1 (40 hex chars) matched standalone;
short SHAs (7..12 hex) require a git-vocabulary context to land --
commit/revision/rev/SHA/hash with `:` or whitespace, git subcommand
invocations (show / log / cherry-pick / rev-parse / checkout / reset /
revert / rebase / bisect / diff / stash) on the SAME LINE as the SHA
(horizontal whitespace only), mail footers (Fixes: / Refs: /
Reverts: / See: / Cc:), GitHub-style `#<sha>` refs, reflog
`HEAD@{<sha>}` references. Full and short forms stay distinct
because we cannot prove they match without the repo. Cap 50.
49 tests.

### 4. receipt refund / void / cancelled-transaction detection — 2793ab1
New `ReceiptFields.refund_amount` populates when the receipt
represents a returned/voided/cancelled transaction. Keyword vocab:
Refund / Refund Amount / Refund Total + Void / Void Sale / Void
Transaction + Cancelled / Cancelled Transaction / Cancellation +
Return / Return Amount / Return Total + Reversal. Stored as a
positive amount regardless of leading minus. Fallback to a top-
level negative Total/Subtotal with an explicit `-` when no keyword.
Per-line negatives stay as discount lines (existing behaviour).
LLM wire format updated. 40 tests.

### 5. code shebang interpreter extraction — edd68e9
New `CodeFields.interpreter` populates when the snippet starts
with a Unix shebang. Direct path takes last segment
(`#!/usr/bin/python3.11` -> `python3.11`); env wrapper takes first
non-flag arg; env `-S` split-args skips flags; env
`--split-string=python3` inline form is handled. Only line 1 is
consulted -- body shebangs are ignored. Leading whitespace before
`#!` is rejected because the kernel exec(2) parser also requires
`#!` at byte 0. LLM wire format updated. 39 tests.
