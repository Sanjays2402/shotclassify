## Trade-offs / design decisions

- Breadcrumbs table-form REQUIRES "Breadcrumbs" header anchor so
  random multi-column tables (cron schedules / API endpoint
  catalogues) never false-positive. JSON form doesn't need the
  anchor because the {"category":..., "message":..., "timestamp":...}
  combo is itself the discriminator.
- Breadcrumb categories curated 30-tag vocab so a row like
  "config /path 10:00" never lands -- "config" isn't a Sentry
  breadcrumb category.
- Ship-to header REQUIRES separator (: or -) at end of line because
  "I'm shipping to my friend later" prose would otherwise false-
  positive on the words "Shipping to" alone.
- Ship-to country catalogue conservative (60 names) because partial
  matches would let prose lines bleed in as country.
- Ship-to terminator vocab includes Bill To / Order # / Subtotal
  so the bill-to block immediately below ship-to never bleeds in.
- Unsafe queries language gating: NARROWER exclude set than
  _NO_IMPORT_LANGUAGES because the language detector commonly tags
  Python+SQL snippets as "sql" when SELECT dominates. We
  intentionally don't exclude "sql" so we still scan those captures
  for f-string / template / concat patterns.
- Unsafe queries f-string regex uses quote-pair variants
  (DQ + SQ + triple-DQ + triple-SQ) so embedded opposite-quote
  chars in the SQL body don't terminate the match.
- S608 ruff fires on intentionally-unsafe test fixtures (the
  strings the detector is verifying it finds). Added # noqa: S608
  on those two test fixtures to keep the gate clean without
  dropping the test cases.
- Trading positions BARE matcher uses a reject-list of common
  prose words (FOR / BY / AT / EST / PST / GMT / UTC / ETA) so
  "5 FOR @ 2.50" prose never tags as a position.
- Trading positions OPTION matcher tried FIRST with span-claim
  defence so option claims aren't double-fired as bare stock.
- Link previews messaging-platform domain reject-list (slack.com /
  discord.com / t.me / etc.) because those appear as client URLs
  in chat metadata far more often than as preview headers.
- Link previews title minimum 3 words rejects sender-name false
  positives sandwiched between domain-shaped lines.
- Link previews URL extraction looks BACKWARD up to 5 lines for
  the most recent http(s):// URL because the preview card sits
  BELOW the message body containing the original URL.
