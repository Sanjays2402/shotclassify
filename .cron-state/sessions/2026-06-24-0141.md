## Slices shipped (5)
1. **`?` keyboard shortcuts help overlay** (4cba253)
   - Linear / Raycast style, opened by `?` outside text inputs OR via a
     header chip with a Keyboard icon + kbd label so mouse users discover it.
   - Catalogue: `web/lib/shortcuts.ts` (7 shortcuts: ⌘K, ?, U, S, C, Esc,
     `g t` -- and `T` for theme cycle added in slice 4).
   - Platform-aware glyph rendering swaps ⌘ for Ctrl on Win/Linux.
   - Multi-stroke sequence tracker (FSM) for `g t` -> scroll to top, with
     a 1200ms window, prefix-trim heuristic so wrong keys don't lock out
     the next sequence, modifier-reset, and non-printable-key reset.
   - HotKeys.tsx rewritten to consult the catalogue before falling through
     to bare-letter nav. Bare-letter combos explicitly reject ANY modifier
     so Cmd-S "save page" / Ctrl-U "view source" never trigger our nav.
   - 15 tests in `lib/shortcuts.test.mts`.

2. **`<EmptyState>` reusable component + filter-aware `/shots` empty** (d7f714e)
   - `web/components/EmptyState.tsx`: eyebrow + felt-green icon well +
     h-display title + body + primary cue-yellow + secondary ghost CTA.
     Variants: panel (default) and bare. Actions accept href OR onClick.
   - `web/lib/empty-state.ts`: pure copy helper -- `hasActiveFilters`,
     `describeFilters` (with 32-char query truncation, single-sided vs
     double-sided date range collapse), `emptyCopyForList(noun, filters)`.
   - Wired into `/shots`: when filters are active, shows "No shots match
     that filter · Active: <summary>" with a one-click "Reset filters"
     CTA. When blank-slate, shows "No shots yet · Ingest a frame" +
     "Try the demo".
   - 9 tests in `lib/empty-state.test.mts`.

3. **`<ConfBadge>` semantic confidence pill** (5537df5)
   - Three sizes (sm/md/lg) and two variants (ghost / solid).
   - `lib/confidence.ts`: pure helpers for tier mapping, aria-label
     composition ("92.0 percent. High confidence."), display formatting
     (clamped), CSS-token-name mapping, and the title-attribute tooltip.
   - Replaces raw `pct()` text spans in `components/Feed.tsx` and the
     `/shots` confidence column. Both spots now share one source of
     truth for tier thresholds + colors + a11y copy.
   - Tier-coloured dot glyph + 1px tier-coloured border + tier-coloured
     text = the tier signal is visually triple-encoded.
   - 8 tests in `lib/confidence.test.mts`.

4. **Dim theme toggle with flash-free pre-paint init** (c218fbc)
   - New `html[data-theme="dim"]` block in `globals.css` re-skins
     chalk-cream to a warm dark backplate. Felt-green and cue-yellow
     accents preserved so the broadcast voice stays recognisable.
     Per-component overrides for `.panel`, `.tbl`, `.eyebrow`, `.kbd`,
     `.conf-bar`, `.btn-ghost`, the header.
   - `lib/theme.ts`: `ThemeMode = "light" | "dim" | "system"`. Pure
     helpers + the `themeInitScript` that runs in `<head>` BEFORE
     hydration so dim users never see a chalk-cream flash.
   - `components/ThemeToggle.tsx`: left-click cycles modes, right-click
     opens a popover menu with all three options, matchMedia listener
     tracks live OS prefers-color-scheme flips.
   - New "T" shortcut in `lib/shortcuts.ts` dispatches a custom
     `shotclassify:theme-cycle` event the toggle listens for.
   - 11 tests in `lib/theme.test.mts` (including 4 init-script tests
     that polyfill window/document/localStorage and assert correct
     attribute writes under each state).

5. **Scroll progress bar + back-to-top FAB** (5e33b04)
   - 3px cue-yellow-to-felt-green gradient progress bar pinned to top
     edge, 80ms width transition smooths rAF-throttled state updates.
   - 44x44 dark FAB with yellow ArrowUp glyph, fades + scales in at
     bottom-right past 600px scroll, with 200ms transitions and

   - Scroll + resize events coalesced through `requestAnimationFrame`,

   - 7 tests in `lib/scroll-progress.test.mts`.

## Tooling fix (0874945, not a feature slice)
- `tests/test_web_api_keys.py` was failing on Node 25 because it used
  bare `node --input-type=module -e <script>` and Node 25 stopped
  accepting direct `.ts` imports via that form. Mirrored the
  `test_webhooks.py` fix and added `--import tsx` to load `.ts`.
- `tests/test_webhooks.py` was failing because (a) it pre-dated the
  SSRF guard so its 127.0.0.1 listener got rejected, and (b)
  `dispatchEvent` gained a `workspace_id` first argument in a later
  tick. Set `SHOTCLASSIFY_WEBHOOK_ALLOW_LOOPBACK=1` and passed
  `DEFAULT_WORKSPACE_ID` through the call. Bumped the post-dispatch
  settle wait from 200ms to 400ms.
- Both failures reproduced cleanly on tick 26's tip (3cc0df1) without
  any of my new commits applied, confirming they were not regressions
  I introduced. Fix shipped this tick because they would have blocked
  every future tick's gate.
