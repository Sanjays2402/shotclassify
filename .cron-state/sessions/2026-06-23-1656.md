## Trade-offs / design decisions
- Lottery: case-sensitive LOTTO/LOTTERY guard for ALL-CAPS POS output.
- Lottery: canonical lookup with prefix-match fallback handles plurals.
- Cancellation: negative lookahead on bare "Non-refundable" so deadline
  form wins over none form when "after Dec 1" follows.
- React: vocab regex restructured because `render()` has non-word `)`.
- React: inner exception accepts optional Uncaught/Unhandled prefix.
- Complexity outlier: strict-greater rule prevents arbitrary tie-breaking.
- Bot messages: Discord BOT regex consumes rest-of-line so body extractor
  picks up real message, not timestamp tail.
- Bot messages: Telegram matcher requires "Bot" suffix to avoid prose
  false-positives.
