1. **e96daf8 — Stripe ID extractor** -> `raw["stripe_ids"]` (cross-category).
   24 typed Stripe object prefixes (cus_ ch_ pi_ inv_ sub_ prod_ price_ acct_
   re_ pm_ seti_ cs_ tr_ po_ txn_ file_ coupon_ promo_ ii_ cn_ txr_ si_ src_

   optional test_ infix + 14..40 alphanumeric chars. Prefixes tried

   Test-mode IDs (cus_test_xyz) recognised explicitly. 50 new tests.

2. **59f2fbb — AWS resource ARN extractor** -> `raw["arns"]` (cross-category).
   Standard 6-segment colon-separated form
   arn:partition:service:region:account:resource. Three partitions
   accepted (aws / aws-cn / aws-us-gov). Region recognises commercial
   (us-east-1), China (cn-north-1), and GovCloud (us-gov-west-1) forms
   or empty for global services. Account recognises 12-digit ID OR
   literal "aws" for AWS-managed IAM policies OR empty for S3 bucket
   ARNs. Resource captured greedily across additional colons and slashes
   so arn:aws:lambda:us-east-1:123:function:foo:1 preserves version
   qualification and CloudWatch log-group multi-colon forms work.
   Trailing punctuation stripped. 41 new tests.

3. **047109a — Discord snowflake ID extractor** -> `raw["discord_ids"]`
   (cross-category). Recognises typed mention forms (<@id>, <@!id>,
   <#id>, <@&id>), jump URLs (discord.com/channels/G/C/M with legacy
   discordapp.com domain), webhook URLs (CAPTURES the ID, DROPS the
   token permanently as a tested security guarantee), and bare 17..19
   digit snowflakes with Discord-context anchor on same or previous
   line. Bare-snowflake matcher REQUIRES the anchor because 17..19
   decimal digit blobs are too easy to confuse with UNIX nanosecond
   timestamps. Pass ordering: typed-mentions -> URLs -> anchored-bare,
   with first-seen kind winning on dedupe. 32 new tests.

4. **3afa78c — TODO author extraction** -> `CodeFields.todo_authors`.
   Mirrors detect_todo_count's 7 ALL-CAPS markers (TODO FIXME XXX HACK
   BUG NOTE OPTIMIZE). Recognises canonical `MARKER(author):` and
   `MARKER(author)` forms across six comment-leader families (// # /* ;
   -- default-hash). Author handle supports plain / with-@ / with-digits
   / with-hyphen / with-underscore / with-period / with-email / full-name
   shapes. Empty-paren rejected. Substring rejection (TODOIST not a
   marker). Dedupe intentionally NOT done because the same author may
   legitimately own multiple TODOs and the count should be accurate.
   LLM wire-format updated. 49 new tests.

5. **6b483c8 — Cash-rounding adjustment** -> `ReceiptFields.rounding`
   (signed). Regulatory adjustment for small-coin scarcity in AU CA NZ
   NO SE CH HU IE NL etc. Six recognised wordings most-specific-first:
   "Rounding Adjustment" / "Cash Rounding" / "Cash Discrepancy" /
   "Rounding" / "Round Down" / "Round Up". New _find_signed_amount_after
   helper captures sign correctly because the existing
   _find_amount_after's [:\\-]? separator class would eat the minus.
   Sign captured both before AND after currency symbol (-$0.02 OR
   $-0.02). Comma-decimal European style (-0,02) supported. Explicit
   0.00 intentionally registers because printing the line at all is a
   useful signal. LLM wire-format updated. 34 new tests.
