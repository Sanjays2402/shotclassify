## Notable design decisions

* **Spring WhiteLabel** detection requires BOTH the literal
  "Whitelabel Error Page" heading AND the "(type=..., status=NNN)"
  summary line so prose mentions in runbooks / templates don't
  false-positive. Path regex restricted to conservative RFC 3986
  path-char set so trailing punctuation in Spring's ", so you are
  seeing" wording doesn't bleed into the captured path. Message
  search iterates by LINE boundaries (not by the summary regex's
  end offset which sits mid-line). HTML closing tags
  `</body></html>` skipped explicitly.

* **Suggested tips** uses two-pass per-line span-claim:
  pct-then-amt pass first, amt-then-pct only over unclaimed
  regions. Prevents phantom cross-pair captures on horizontal
  table rows like "15% 1.80   18% 2.16" where a naive amt-then-pct
  pass would stitch "1.80 18%" as a bogus pair. Requires AT LEAST 2
  distinct pairs because a lone pair is the customer's actual tip.

* **Points earned** balance-vs-earn distinction is the critical
  safety property. 11-token disqualifier vocabulary on the same line
  rejects "Total Points: 1245" while accepting "Points Earned: 25".
  Trailing negative-lookahead on `[.,]?digit` blocks 10-digit
  partial matches like 10000000 -> 1000000.

* **GraphQL** first-error-isolation via bracket-depth tracker with
  quoted-string awareness so multi-error arrays correctly attribute
  code+locations+path to the FIRST entry. Detection requires errors
  array + message field + one discriminator
  (locations/path/extensions/graphql/apollo/mutation/subscription/
  query) so generic REST errors don't false-positive.
  Placed at the TOP of parse_error_text because GraphQL JSON can
  contain JS-style stack traces in extensions.exception.stacktrace
  that the Node branch would otherwise steal.

* **TODO tickets** per-line span-claim discipline so JIRA-1234
  isn't mis-tagged as a slug or its trailing digits as hash-num.
  Three matchers in priority order: JIRA (PROJECT-NUMBER) first,
  then hash-slug (#identifier-NUMBER), then hash-num (#NUMBER).
  Coexists with todo_authors so TODO(alice): #1234 populates both.
