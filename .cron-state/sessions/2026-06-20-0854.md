## Shipped (5 commits on feature/autoship)

1. **29aedf0** `feat(extract/chat): timestamp parsing from message lines`
   - New `parse_timestamp()` helper handles ISO8601 (full TZ /
     seconds optional), 12-hour AM/PM (case-insensitive,
     `a.m.` / `am` / `AM` all accepted), and bare 24-hour clocks.
   - AM/PM normalised to 24h `HH:MM` so dashboards sort
     lexicographically; ISO returned verbatim so the screenshot's
     timezone is preserved.
   - `enrich_chat` now backfills `messages[i]["time"]` and strips
     the stamp + adjacent separator from the body. Works on both
     freshly-parsed sender lines and LLM-supplied messages that
     lack a time field.
   - 20 tests covering each shape, edge cases (12 AM -> 00,
     12 PM -> 12, garbled clocks), ISO-wins-over-bare, and the
     enrich pipeline integration.

2. **120b0ae** `feat(extract/receipt): discount, coupon, promo extraction`
   - New optional `ReceiptFields.discount`. Recognises Discount,
     Coupon, Promo, Promo code, Savings, Loyalty, Rewards labels.
   - Last-occurrence wins so an "Available coupons" header at
     the top of the receipt cannot override the line the cashier
     actually rang up.
   - Items parser now skips these labels so the discount no longer
     surfaces as a phantom $2.00 line item.
   - Wire-format pass in `client.py` updated (and fixed two prior
     omissions: `tip` and `payment_method` were missing too).
   - 15 tests including the full receipt scenario where discount /
     tip / tax coexist, plus a regression-guard test that documents
     the shared amount regex does NOT accept a sign in front of
     the digit run.

3. **b9ecf39** `feat(extract/receipt): derive tip_percent from tip and subtotal`
   - New optional `ReceiptFields.tip_percent`. Computed as
     `tip / subtotal * 100`, rounded to one decimal because OCR
     noise makes two decimals spurious.
   - Falls back to `tip / (total - tip)` when only the total is
     available (tax distortion accepted because the bucket -- 15 /
     18 / 20 / generous -- is still useful for dashboards).
   - Returns None on no-tip, zero-tip (so charts don't get a 0.0%
     bucket from a printed "Tip 0.00"), and no-base cases.
   - `enrich_receipt` recomputes tip_percent against the merged
     values when the caller didn't already supply one, so an LLM
     that saw a "Tip 20%" header is still trusted over arithmetic.
   - 14 tests.

4. **177da1d** `feat(extract): cross-category URL extractor into raw["urls"]`
   - New `shotclassify_extract.urls` module exporting
     `extract_urls(text) -> list[str]`.
   - Wired into `enrich()` so EVERY category populates
     `ExtractedFields.raw["urls"]` -- URLs show up in errors,
     code, receipts, chats, everywhere.
   - Conservative regex: `http(s)://` only (bare domains have too
     many OCR false positives), stops at whitespace / quotes /
     closing brackets, trims trailing sentence punctuation
     iteratively (so `...).` collapses cleanly), de-dupes preserving
     first-seen order, capped at 50.
   - 16 tests including the parameterised per-category integration,
     markdown-link extraction, query+fragment preservation, and
     the empty-list / None / no-key edge cases.

5. **7f8dcaa** `feat(extract/chat): hashtag and mention extraction`
   - New optional `ChatFields.hashtags` and `ChatFields.mentions`.
   - Hashtags require leading letter / underscore (rejects #123
     issue refs), preserve case (#OpenAI vs #openai matter),
     capped at 50.
   - Mentions are word-boundary anchored on the left so
     `foo@bar.com` (email) does NOT produce a phantom `@bar`,
     de-duped case-insensitively (Slack / Twitter treat
     @Sanjay == @sanjay), trim trailing sentence punctuation,
     and surface `@channel` / `@here` / `@everyone` up front
     with prefix intact.
   - Caller-supplied lists are preserved and unioned with parsed
     values (caller first). Wire-format passes the new fields
     through `client.py`.
   - Also amended a tiny import-sort cleanup in
     `extract/__init__.py` that the new exports made necessary.
   - 13 tests including the combined timestamp + hashtag + mention
     integration scenario.

## Decisions worth noting next tick

- I started leveraging `ExtractedFields.raw[<key>]` for cross-category
  enrichments this tick (URLs). It's a JSON dict, already persisted
  by storage, no migration needed -- a great pattern for future
  features like the file-path extractor I dropped in the backlog.
- Receipt extraction is reaching the point where the underlying
  `_find_amount_after` regex is doing a lot of work. Future features
  that touch it (e.g. signed amounts for refunds) should consider a
  dedicated regex rather than widening the shared one.
- Chat enrichment is now substantial enough that the chat.py file
  is ~250 lines. If it grows much more, splitting timestamp / tag /
  mention helpers into sibling modules would be reasonable. Today
  it's still small enough to read top-to-bottom.
- The classify client wire-format mapping in `client.py` keeps
  getting kwarg holes when new fields are added. Next tick I might
  refactor it to do a `model_validate(ch)` against the Pydantic
  model so we stop missing fields.
