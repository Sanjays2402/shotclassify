## Tricky bits / learnings

- The .NET frame regex initially used `[^:]+` to capture the file path,
  which broke on Windows drive letters (`C:\src\App.cs`). Switched to
  `\S+?` (non-greedy, no whitespace) and it survives the drive-letter
  colon while still stopping before `:line N`.
- pytest's `>` indicator can mark ANY source line, not just `assert`
  lines (a `>       parse("")` call that raised is also valid). Initial
  regex only matched `>\s*assert\s+...` which missed half of real
  pytest failures. Generalised to capture any `>` line, then strip a
  leading `assert ` keyword in the helper.
- The receipt line-item parser had a keyword-skip filter that stripped
  any line containing "promo", "discount", "coupon" etc. — which would
  have killed `Promo: BOGO 50% off Latte 4.00`. Added a pre-filter that
  checks for `\d+% off` BEFORE the keyword skip so the discount line
  escapes the filter.
- Ruff I001 (`Import block is un-sorted or un-formatted`) wants no
  blank line between `from __future__` and the first non-future import.
  Test file docstrings count toward the "import block placement" so
  the docstring + blank line + `from __future__` + blank line +
  `import X` pattern triggers I001. Removed the blank line between
  the future import and the first regular import to fix.
